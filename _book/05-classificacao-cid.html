6  Classificação por CID

Este capítulo apresenta a primeira camada decisória do pipeline: a classificação sindrômica por CID. Nesta proposta, o CID é considerado uma camada soberana. Quando o CID está presente e é compatível com uma das síndromes definidas, ele prevalece na classificação final.

6.1 Princípio da camada CID

A classificação por CID é direta e auditável. Ela parte de uma tabela de mapeamento entre códigos CID e síndromes febris.

ImportantRegra de soberania

Se o CID principal estiver presente e for mapeado para uma síndrome, a classificacao_cid será considerada a classificação final prioritária. As camadas regex e LLM continuarão sendo calculadas, mas servirão para auditoria, comparação e identificação de divergências.

6.3 Carga da base padronizada

Nesta etapa, a base sintética gerada no capítulo anterior é carregada e padronizada novamente para garantir reprodutibilidade do pipeline.

Ver código 
if (!file.exists("data/atendimentos_sinteticos.csv")) {
  save_synthetic_attendances(
    path = "data/atendimentos_sinteticos.csv",
    n = 700,
    seed = 123
  )
}

dados_raw <- readr::read_csv(
  "data/atendimentos_sinteticos.csv",
  show_col_types = FALSE
)

mapping_sintetico <- list(
  record_id = "record_id",
  data_atendimento = "data_atendimento",
  unidade = "unidade",
  idade = "idade",
  sexo = "sexo",
  cid = "cid",
  queixa = "queixa",
  anamnese = "anamnese"
)

dados <- normalize_input_data(
  df = dados_raw,
  mapping = mapping_sintetico
)

dados <- dados |>
  dplyr::left_join(
    dados_raw |>
      dplyr::select(
        record_id,
        territorio,
        bairro,
        cid_nome,
        sindrome_esperada_sintetica,
        cid_status_sintetico,
        febre_negada_sintetica
      ),
    by = "record_id"
  )

6.9 Persistência em DuckDB

A classificação por CID é persistida no banco local para auditoria e uso nas etapas seguintes.

Ver código 
con <- connect_duckdb()

  con = con,
  dados_cid = dados_cid,

6.10 Interpretação

A camada CID oferece uma classificação altamente rastreável, mas depende da qualidade do preenchimento e da especificidade do código utilizado. Por isso, ela será preservada como camada soberana, enquanto as camadas de regex e LLM atuarão como mecanismos complementares para:

  • classificar registros sem CID informativo;
  • auditar discordâncias entre texto e CID;
  • apoiar revisão de casos divergentes;
  • subsidiar análises de qualidade dos registros.
